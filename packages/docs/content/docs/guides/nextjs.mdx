## Configure Next.js

Use the `withOpenTui` plugin to configure webpack automatically:

```ts title="next.config.ts"
import { withOpenTui } from "opentui-web/next-plugin"

export default withOpenTui("../path/to/opentui")
```

The first argument is the path to the opentui monorepo root. You can pass
additional Next.js config as a second argument:

```ts title="next.config.ts"
import { withOpenTui } from "opentui-web/next-plugin"

export default withOpenTui("../path/to/opentui", {
  reactStrictMode: true,
  // ... other Next.js config
})
```

If you provide your own `webpack` function, `withOpenTui` will chain it —
your config runs first, then opentui aliases and shims are applied on top.

</Step>
<Step>

## Create Your Component

Mark your page as a client component and use `OpenTuiCanvas`:

```tsx title="app/page.tsx"
"use client"

import { OpenTuiCanvas } from "opentui-web"

function TuiDemo() {
  return (
    <box border borderStyle="rounded" padding={1}>
      <text fg="#a3be8c">Hello from Next.js!</text>
    </box>
  )
}

export default function Home() {
  return (
    <OpenTuiCanvas style={{ width: "100vw", height: "100vh" }}>
      <TuiDemo />
    </OpenTuiCanvas>
  )
}
```

## SSR Safety

<Callout type="success">
`OpenTuiCanvas` is SSR-safe by default. On the server it renders an empty
container div. On the client it hydrates and creates the canvas. No
`dynamic(() => import(...), { ssr: false })` wrapper is needed.
</Callout>

## Key Differences from Vite

| Concern | Vite | Next.js |
|---------|------|---------|
| Plugin | `opentuiWebPlugin()` in vite.config | `withOpenTui()` in next.config |
| `node:` imports | Handled by resolveId hook | `NormalModuleReplacementPlugin` |
| CJS modules | Virtual module redirects | Webpack handles natively |
| Top-level await | Supported by default | Enabled by plugin |
| Node.js builtins | Shimmed via resolveId | Shimmed via `resolve.alias` with `$` suffix |
